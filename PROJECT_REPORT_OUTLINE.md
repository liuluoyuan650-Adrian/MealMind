# MealMind 项目报告提纲

## 1. 项目背景

现代大学生日常吃饭经常面临选择困难：外卖和食堂菜品数量多，但用户真实需求往往是自然语言表达的，例如“预算 30 以内”“不要辣”“想吃清淡热乎的”“减脂但要吃饱”。传统菜单筛选需要用户手动选择多个条件，效率较低。

本项目开发 MealMind 个性化菜品推荐系统，面向校园生活场景，通过自然语言理解技术解析用户吃饭需求，并推荐具体菜品。

## 2. 问题描述与实际需求

系统需要解决的问题：

- 用户不想填写复杂表单，希望直接用自然语言描述需求。
- 用户需求包含多个约束，例如预算、口味、忌口、健康需求和情绪状态。
- 推荐结果需要可解释，不能只给出菜名。
- 系统需要支持用户继续补充条件并重新推荐。

目标用户：

- 大学生
- 校园周边就餐用户
- 有预算、忌口、减脂或选择困难需求的用户

## 3. 主要功能

- 自然语言需求输入
- 用餐意图识别
- 槽位抽取
- 菜品数据库检索与过滤
- 100 分制推荐排序
- 推荐理由生成
- 多轮对话条件更新
- 前端可视化展示

## 4. 技术方案

技术选型：

- Streamlit：快速构建可交互 Web 演示界面。
- pandas：读取和处理菜品 CSV 数据集。
- scikit-learn：实现 TF-IDF + Logistic Regression 意图分类模型。
- 正则表达式：抽取预算、人数等结构化信息。
- 关键词词典：抽取口味、忌口、健康需求、情绪场景。
- 规则打分算法：实现可解释菜品推荐。

系统架构：

用户输入 → 意图识别 → 槽位抽取 → 硬性过滤 → 规则打分 → 推荐理由生成 → 前端展示

## 5. 关键算法 / 模型原理

### 5.1 意图识别

使用字符级 TF-IDF 将用户输入转为文本向量，再使用 Logistic Regression 进行分类。可识别清淡餐、减脂餐、夜宵、高蛋白餐、预算餐、治愈餐、不辣餐、多人聚餐菜品、快速饱腹餐、甜食饮品推荐等意图。

### 5.2 槽位抽取

预算和人数使用正则表达式抽取；口味、忌口、健康需求和场景使用关键词词典匹配。系统将自然语言转换为结构化约束。

### 5.3 推荐排序

系统先执行硬性过滤，例如用户明确“不吃辣”时直接排除辣度大于 0 的菜品。然后执行 100 分制规则打分：

- 忌口安全：20 分
- 预算匹配：20 分
- 口味匹配：15 分
- 场景匹配：15 分
- 健康需求：10 分
- 饱腹感：10 分
- 温度偏好：8 分
- 评分与偏好补充：若干加分

## 6. 创新点

1. 面向生活场景的多约束自然语言菜品推荐。
2. 将忌口作为硬约束，避免推荐明显不合理的菜品。
3. 推荐结果可解释，展示推荐分和匹配理由。
4. 支持情绪和场景感知，例如疲惫、治愈、减脂、夜宵。
5. 支持多轮补充条件，用户可以继续说“不要面，想吃米饭”。

## 9. 思考与总结

遇到的问题：

- 自然语言表达方式多样，单纯关键词匹配容易漏掉隐含语义。
- 真实餐饮数据获取困难，因此本项目使用模拟校园菜品数据集。
- 多约束推荐中，硬性约束和软偏好需要区分。

解决方案：

- 使用意图分类模型处理整体场景。
- 用正则和词典抽取明确槽位。
- 忌口先硬性过滤，其他偏好参与加权打分。

局限性：

- 数据集规模有限。
- 暂未接入真实外卖或地图 API。
- 对复杂长句和隐含语义的理解仍有限。

未来改进：

- 扩充真实校园菜品数据。
- 引入句向量模型提升语义匹配能力。
- 接入大模型 API 润色推荐理由。
- 增加用户历史偏好学习。
